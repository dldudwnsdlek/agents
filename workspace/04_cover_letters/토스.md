## 지원 동기

토스 기술블로그에서 Ads ML 파이프라인이 후보군을 Two-tower·Lookalike로 좁히고, CTR 예측에 FM·DeepFM·DCN을 쓰며, 동적입찰은 DQN 강화학습으로, 소재 최적화는 생성형 AI로 푼다는 글을 읽으며 이 팀에 지원하고 싶다는 생각이 분명해졌습니다. Targeting → Filtering(Pacing/Frequency Capping) → Ranking(eCPM)으로 이어지는 노출 구조는, 제가 추천 연구에서 다뤄 온 '후보 생성과 랭킹'의 문제를 광고라는 대규모 실서비스에 그대로 옮겨 놓은 형태였습니다.

특히 CTR 예측의 핵심 모델군인 DeepFM·DCN은 제가 직접 구현한 AutoInt와 같은 가계입니다. 저는 AutoInt를 논문만 보고 TensorFlow 커스텀 레이어로 짜 본 적이 있어, 토스 Ads의 모델 스택에 빠르게 합류하고 곧바로 개선 실험을 제안할 수 있다고 판단했습니다. 첫 연간 흑자에 이어 광고 사업 월 매출이 100억 원대를 넘고 미국 증시 IPO를 준비하는 단계에서, 추천·광고 모델의 한 줄 개선이 곧 매출 지표로 이어지는 환경 또한 제가 일하고 싶은 무대입니다.

## 직무 역량 및 경험

**AutoInt 기반 CTR 추천 — 논문을 코드로 (Situation·Task·Action·Result)**
추천 모델의 성패를 가르는 특성 상호작용을 사람의 직관이 아니라 Self-Attention으로 학습시키고 싶었습니다(S). 목표는 AutoInt 논문을 재현하는 데 그치지 않고 실시간 추천이 동작하는 형태까지 만드는 것이었습니다(T). MovieLens-1M 100만 건에서 14개 특성을 통합하고, AutoInt를 Embedding → Multi-Head Self-Attention 3층 + Residual → Sigmoid 구조로 TensorFlow 커스텀 레이어를 직접 구현했으며, 학습된 모델을 Streamlit으로 감싸 실시간 Top-10을 반환하는 데모까지 연결했습니다(A). 그 결과 논문 구조를 동작하는 추천기로 옮기는 전 과정을 혼자 수행했고, 어텐션 가중치로 어떤 특성 조합이 클릭에 기여하는지 해석할 수 있는 형태를 확보했습니다(R). DeepFM·DCN으로 확장할 때, 같은 계열 모델을 이미 손으로 구현해 본 경험이 학습 속도를 끌어올릴 것입니다.

**후보 생성부터 랭킹까지 — 추천 모델의 폭 (제1저자·은상)**
Two-tower가 임베딩 유사도로 후보를 만들 듯, 저는 그래프 기반 후보 생성도 다뤘습니다. Melon 플레이리스트 14만 건에 멤버-그룹 지식그래프 1,612개 노드를 GCN으로 학습시켜 LightGCN과 순차 융합하는 구조를 설계하고, 데이터 수집부터 모델 설계·구현·평가까지 직접 수행해 Recall@20·NDCG@20에서 기존 LightGCN 대비 향상을 확인했습니다. 이 연구로 한국디지털콘텐츠학회 추계종합학술대회에서 은상을 받았습니다(R). 여기에 NCF·BPR, 그리고 Amazon 리뷰 791만 건을 DistilBERT로 감정분류한 뒤 NCF와 결합한 하이브리드 추천(Hit Rate@10 0.667)까지 다뤄, 후보 생성·랭킹·협업필터링을 가로지르는 추천 문제에 빠르게 적응할 수 있습니다.

대규모 데이터를 끝까지 처리하는 파이프라인 경험도 갖췄습니다. 791만 건 감정분류와 금융결제원 FDS의 45개+ 파생 Feature 설계·날짜 분할 검증 등, 데이터 전처리부터 모델·평가까지 End-to-End로 완수해 온 점은 광고처럼 데이터 규모가 큰 환경에서도 기여로 이어질 것입니다.

## 입사 후 포부

입사 초기에는 Ads ML 파이프라인과 랭킹 모델, A/B 테스트 흐름을 빠르게 익혀 CTR·eCPM 예측 모델의 오프라인 지표 개선부터 기여하겠습니다. 제가 강한 CTR/추천 모델링과 논문 구현 능력에, 토스가 쌓아 온 동적입찰(DQN)·생성형 소재 최적화 같은 영역을 더해, 실험을 설계하고 검증해 서비스 지표로 증명하는 ML 엔지니어로 자리잡고자 합니다. 나아가 추천과 광고가 만나는 지점에서, 어텐션 기반 모델 해석 경험을 살려 성능과 설명가능성을 함께 끌어올리는 데 기여하겠습니다.
