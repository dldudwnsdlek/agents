# 자기소개서 작성 (cover-letter-write)

## 언제 쓰나 (triggers)
- 채용 파이프라인의 **4단계**로 실행될 때
- 특정 회사·공고에 맞춘 자기소개서가 필요할 때

## 입력
- `workspace/03_analysis.md` — 기업 분석(필수)
- `workspace/02_evaluated.json`의 `passed` — 회사·직무·URL(필수)
- `PORTFOLIO.md` — 지원자의 실제 경험·성과(필수, 사실의 원천)

## 작성 원칙
- **STAR 구조**(상황-과제-행동-결과), 결과는 가능하면 수치로.
- 회사 키워드(인재상·기술 스택·"우리가 찾는 사람")를 자연스럽게 호응.
- 한국식 축: **지원동기 / 직무 역량(경험) / 입사 후 포부**.
  공고에 문항이 명시돼 있으면 그 문항에 정확히 답하고 글자 수 제한을 지킨다(±5%).
- 클리셰 금지: "어릴 적부터", "열정 하나만큼은", "귀사의 무궁한 발전" 등.
- **포트폴리오에 근거 없는 경험·수치·기술은 쓰지 않는다.** 과장 의심 부분은 `<!-- 확인 필요 -->`.

## 절차 (steps)
1. `passed` 회사 목록을 읽고, 회사별로 `03_analysis.md`의 해당 섹션과 `PORTFOLIO.md`를 결합한다.
2. 회사마다 어필 포인트 Top 3를 정한다(분석가가 짚은 회사 니즈 ↔ 포트폴리오 강점 매칭).
3. 회사별 자소서를 작성해 `workspace/04_cover_letters/<회사명>.md`에 저장한다.
4. 각 파일 머리말에 **지원 회사·직무·어필 포인트 3개 메모**를 단다.

## 출력 형식 — `workspace/04_cover_letters/<회사명>.md`
```markdown
<!-- 지원: [회사명] / [직무명] / 어필포인트: 1) … 2) … 3) … -->

## 지원 동기
...(회사 사업/문화에 포트폴리오를 연결)...

## 직무 역량 및 경험
...(STAR로 1~2개 핵심 경험, 수치 포함)...

## 입사 후 포부
...(회사의 방향성과 본인 성장 계획 연결)...
```

## 흔한 함정 (pitfalls)
- 템플릿 복붙 금지 — 회사마다 새로 직조한다.
- 지원 여부를 결정하지 않는다(결정자의 몫). "지원할 경우 최고의 자소서"를 만든다.
- 경력·수상·기술 날조 금지. 근거 없으면 쓰지 않거나 `<!-- 확인 필요 -->` 표시.
- 문항별 글자 수 제한이 있으면 반드시 지킨다.

## 성공 기준 (verify)
- `04_cover_letters/`에 `passed` 회사 수만큼 파일이 생성됨.
- 각 파일이 지원동기·직무역량·입사후포부(또는 공고 문항)를 충족.
- 클리셰·근거 없는 과장이 없다(있으면 주석 표시).
