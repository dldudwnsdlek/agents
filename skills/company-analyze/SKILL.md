# 기업 심층 분석 (company-analyze)

## 언제 쓰나 (triggers)
- 채용 파이프라인의 **3단계**로 실행될 때
- 특정 기업(들)에 대한 지원자 관점의 분석이 필요할 때

## 입력
- `workspace/02_evaluated.json`의 `passed` 목록(필수)
- `PORTFOLIO.md` — 지원자 관점 핏 판단용(필수)

## 분석 프레임 (회사별 5축)
1. **개요** — 업종, 주력 제품/서비스, 설립연도, 인원 규모, 본사 위치
2. **사업/성장성** — 비즈니스 모델, 최근 투자/매출/시장 위치, 성장 단계
3. **조직/문화** — 기술 스택, 개발 문화, 일하는 방식(기술블로그·채용페이지 단서)
4. **평판/리스크** — 강점과 우려점(재무 안정성·이직률·논란 등; 익명 커뮤니티는 '단서'로만)
5. **지원자 관점 핏** — 이 포트폴리오의 사람에게 주는 기회와 리스크

## 절차 (steps)
1. `02_evaluated.json`의 `passed` 회사 목록을 읽는다.
2. 회사별로 `web` 툴셋으로 공식 사이트·채용페이지·기술블로그·뉴스·투자정보를 조사한다.
3. 5축 프레임으로 정리하고, **핵심 주장마다 출처 URL**을 단다. 확인 불가 정보는 `(미확인)`.
4. 회사별 **한 줄 평 + 별점(★~★★★★★) + 핵심 기회/리스크 3개**를 요약한다.
5. 결과를 `workspace/03_analysis.md`에 저장한다(회사별 섹션 + 머리말 요약표).

## 출력 형식 — `workspace/03_analysis.md`
```markdown
# 기업 분석 리포트 (YYYY-MM-DD)

## 요약표
| 회사 | 직무 | 별점 | 한 줄 평 | 핵심 리스크 |
|---|---|---|---|---|
| ... | ... | ★★★★ | ... | ... |

## [회사명] — [직무명]
- **개요**: ...
- **사업/성장성**: ... (출처: URL)
- **조직/문화**: ... (출처: URL)
- **평판/리스크**: 강점 … / 우려 … (출처/단서)
- **지원자 관점 핏**: 기회 … / 리스크 …
- **별점**: ★★★★ · **한 줄 평**: ...
```

## 흔한 함정 (pitfalls)
- 자소서를 쓰거나 지원 여부를 결정하지 않는다 — "판단 재료"만 만든다.
- 출처 없는 단정 금지. 익명 커뮤니티 글은 무비판적으로 인용하지 말고 '단서'로만 표시.
- 추측과 사실을 섞지 않는다. 공개 정보가 부족하면 그렇게 적는다.

## 성공 기준 (verify)
- `03_analysis.md`에 `passed`의 **모든 회사**가 5축으로 정리됨.
- 핵심 주장에 출처가 있거나 `(미확인)`으로 표시됨.
- 머리말 요약표가 존재한다.
